❓ TROUBLESHOOTING
════════════════════════════════════════════════════════════════

"Could not connect to API"
  → Make sure Terminal 1 (API) is still running
  → Check port 8000 is not blocked
  → Restart both services

Port Already in Use (8000 or 8501)
  → Change port in startup script
  → Or kill process using that port

Movies Not Loading
  → Check Data/tmdb_movies_cleaned.csv exists
  → Verify models/ files present
  → Check console for error messages

Slow Response on First Request
  → COMPLETELY NORMAL!
  → Model loading takes 5-10 seconds
  → Wait a moment and try again

🎯 KEY API ENDPOINTS
════════════════════════════════════════════════════════════════

GET /api/movies/popular
  Returns: List of popular movies
  Example: http://127.0.0.1:8000/api/movies/popular?limit=20

GET /api/movies/search?q=query
  Returns: Movies matching search
  Example: http://127.0.0.1:8000/api/movies/search?q=avatar

GET /api/movies/{movie_id}
  Returns: Detailed movie information
  Example: http://127.0.0.1:8000/api/movies/19995

GET /api/recommendations/{movie_id}
  Returns: Recommended similar movies
  Example: http://127.0.0.1:8000/api/recommendations/19995

POST /api/recommendations/by-title
  Returns: Recommendations by movie title
  Example: http://127.0.0.1:8000/api/recommendations/by-title?title=Avatar

Full interactive docs: http://127.0.0.1:8000/docs


🤖 RECOMMENDATION ALGORITHM
════════════════════════════════════════════════════════════════

How It Works:

1. SBERT Encoder (70% weight)
   - Analyzes movie overviews with sentence transformers
   - Finds semantic similarity (plot, theme, mood)
   - Returns top 50 semantic matches

2. TF-IDF Matcher (30% weight)
   - Analyzes genres, cast, director, keywords
   - Finds metadata similarity
   - Returns top 50 metadata matches

3. Hybrid Ranking
   - Combines both approaches
   - Score = 0.7 × semantic + 0.3 × metadata
   - Ranks and returns top N results

Result: More intelligent recommendations than single-method approaches!
